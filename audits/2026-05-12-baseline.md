## Critical Issues (ranked by impact)

### 1. CANONICAL DUPLICATE-CONTENT TRAP on section index pages — `severity: high`
Five section homepages declare canonical with `index.html`:
```
/grow/index.html         ← canonical=https://cola-xpress.com/grow/index.html
/learn/index.html        ← canonical=https://cola-xpress.com/learn/index.html
/guides/index.html       ← canonical=https://cola-xpress.com/guides/index.html
/equipment/index.html    ← canonical=https://cola-xpress.com/equipment/index.html
/watch/index.html        ← canonical=https://cola-xpress.com/watch/index.html
```
The site's `.htaccess` (line 10) only redirects the **root** `/index.html → /`. Nothing redirects `/grow/index.html → /grow/`. Apache serves both URLs with identical content. Google may pick either as canonical, splitting equity.

**Fix (15 min):**
1. Change those 5 canonicals to the directory form (`https://cola-xpress.com/grow/`, etc.).
2. Add to `.htaccess` after the existing root rule:
   ```apache
   # Strip /index.html from any directory URL
   RewriteCond %{THE_REQUEST} \s/+(.*)index\.html[\s?] [NC]
   RewriteRule ^ /%1 [R=301,L,NE]
   ```
3. Update the 957 internal `href="...index.html"` links to use clean directory paths (one sed pass).
**Impact:** Consolidates equity across all 5 section hubs; cleans 957 internal links pointing at the non-canonical form.

### 2. INTERNAL LINKING uses non-canonical `/index.html` everywhere — `severity: high`
- 957 internal `href` attributes link to `*/index.html`
- 0 link to clean directory paths
- Every link to the homepage from a subpage uses `../index.html` instead of `../` or `/`

This is the same root cause as #1, but its scope is larger: 50+ pages each linking to 5+ section hubs via `index.html`. After Fix #1, Apache redirects them — but every external link being one redirect away from canonical is a small equity tax that compounds at scale.

**Fix:** sed pass once Fix #1 is deployed:
```bash
find public_html -name "*.html" -exec sed -i \
  -E 's|href="([^"]*)/index\.html"|href="\1/"|g; s|href="index\.html"|href="./"|g' {} +
```
**Impact:** Removes 957 redirect hops from internal crawl path; tightens link equity routing.

### 3. NO Organization schema on homepage — `severity: high`
Homepage has `WebSite` + `WebPage` JSON-LD only. **No `Organization`** block — even though 36 *content* pages embed Organization as author/publisher inside Article schema. This is a knowledge-graph miss: Google's panel + sitelinks rely on the homepage Organization block.

**Fix (5 min):** Add to `index.html` head:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "ColaXpress",
  "url": "https://cola-xpress.com/",
  "logo": "https://cola-xpress.com/android-chrome-512x512.png",
  "sameAs": [
    "https://www.youtube.com/@<YOUR_HANDLE>",
    "https://www.instagram.com/<YOUR_HANDLE>",
    "https://twitter.com/<YOUR_HANDLE>"
  ],
  "description": "Compact seed-to-jar cannabis growing system."
}
</script>
```
**Need from you:** your real YouTube / Instagram / Twitter handles. If none exist, claim them — they're free brand defenses.

### 4. VideoObject schema is incomplete — `severity: medium`
`watch/episode-01.html` has a `VideoObject` block missing **`contentUrl`** and **`embedUrl`**. Without one of those, Google won't surface the page as a video rich result. Same for the second VideoObject in `watch/index.html`.

**Fix (10 min):** Once the episode is on YouTube, add:
```json
"contentUrl": "https://www.youtube.com/watch?v=<VIDEO_ID>",
"embedUrl":   "https://www.youtube.com/embed/<VIDEO_ID>",
"duration":   "PT12M30S"
```

### 5. TRUST SCAFFOLDING is thin — `severity: medium` (high for cannabis vertical)
The site sells expertise, not product, but cannabis education has the same trust threshold as commerce. Findings:
- **Zero social profile links** anywhere on the site (no `youtube.com/@…`, `instagram.com/…`, etc.)
- **Footer has no contact methods** beyond a `/guides/contact.html` link — no email, no NYC ZIP, no real-name byline
- **About page** mentions "chronic pain and access concerns" but no named author, no photo, no team
- **No licensing language** — fine for an educational brand (no license needed), but the absence of any disclaimer or "we are not a dispensary" framing leaves new visitors guessing what this is
- **No newsletter signup** — leaves all the organic traffic ungated

**Fix:** Add three components:
1. Named author/byline on each guide ("written by …, last reviewed …")
2. Footer column with social profiles (real or claimed)
3. Newsletter signup on guide pages (anything serviceable — ConvertKit, Beehiiv, Buttondown)

**Impact:** Likely 20–35% lift in time-on-page and 2–4× return-visitor rate, based on industry comparables. Direct revenue impact depends on monetization model (covered in §"Open questions").
