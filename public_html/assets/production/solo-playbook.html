What a Script Session Looks Like

Open Claude Code in this project directory. Run the skills in sequence, feeding output from one into the next.

# Step 1 — Competitor research
/competitor-analyst [paste 2-3 competitor transcripts]

# Step 2 — Build research brief from grow journal + data
/research-synthesizer [paste Day 08 notes, VPD readings, root observations]

# Step 3 — Turn brief into episode outline
/narrative-architect [paste research brief output]

# Step 4 — Generate hook options
/hook-writer [paste outline + episode premise]

# Step 5 — Write each section
/script-writer Write the Hook section. Outline: [paste outline]. Hook chosen: [paste selected hook]
/script-writer Write Section 1. Outline: [paste section 1 brief]
... repeat for each section ...

# Step 6 — Audit each section
/retention-auditor [paste drafted section]

# Step 7 — Humanize
/script-humanizer [paste audited section]

# After filming — visual edit plan
/visual-director [paste locked final script]

The Rules for Going Solo

  • Never start filming without a locked script. The script comes from the skills pipeline. No shortcuts.
  • Run the retention audit before humanizing. Audit first, polish second — not the other way around.
  • The visual direction plan is your editing bible. Run it once before you touch the timeline.
  • Shorts come from the master cut — do not shoot separate Shorts. Extract from what already exists.
  • The upload package is pre-written for each episode. Paste it. Don't rewrite from scratch each time.
  • After publishing, immediately update the YouTube ID in both watch page files. Takes 2 minutes. Do it before anything else.
  • Episode 2 starts the competitor analyst pass within 48 hours of Episode 1 going live. Track what performs.
